# Itelma Frontend

Фронтенд-приложение, разработанное с использованием **Vite + TypeScript** и архитектурного подхода **Feature-Sliced Design (FSD)**.
Проект предназначен для модульной и масштабируемой разработки с акцентом на удобство поддержки и повторного использования компонентов.

---

## 🚀 Технологии

* ⚡ [Vite](https://vitejs.dev/) — быстрый сборщик фронтенд-приложений
* 🟦 [TypeScript](https://www.typescriptlang.org/) — строгая типизация
* 🎨 [UnoCSS](https://uno.antfu.me/) — атомарный CSS-фреймворк
* 🧩 Feature-Sliced Design (FSD) — методология организации кода
* 🐳 Docker — контейнеризация приложения

---

## 📂 Структура проекта

```
src/
 ├─ app/        # Инициализация приложения, провайдеры
 ├─ assets/     # Статические ресурсы (иконки, изображения, стили)
 ├─ entities/   # Сущности предметной области
 ├─ features/   # Отдельные функции (логин, фильтры и т.п.)
 ├─ pages/      # Страницы приложения
 ├─ shared/     # Общие модули (UI, либы, утилиты, API)
 └─ widgets/    # Крупные интерфейсные блоки
```

## ⚙️ Установка и запуск

### 1. Клонирование проекта

```bash
git clone https://github.com/your-repo/itelma-front-main.git
cd itelma-front-main
```

### 2. Установка зависимостей

```bash
npm install
```

### 3. Запуск в режиме разработки

```bash
npm run dev
```

Приложение будет доступно по адресу: `http://localhost:5173`

### 4. Сборка для продакшена

```bash
npm run build
```

Собранные файлы будут находиться в директории `dist/`.

### 5. Локальный просмотр сборки

```bash
npm run preview
```

---

## 🐳 Запуск в Docker

Сборка образа:

```bash
docker build -t itelma-front .
```

Запуск контейнера:

```bash
docker run -p 80:5173 itelma-front
```

После этого приложение будет доступно по адресу `http://localhost`.

## 📖 Методология

Проект построен по принципам **Feature-Sliced Design (FSD)**:

* `app` — точка входа, настройка окружения
* `processes` *(опционально)* — долгоживущие процессы
* `pages` — маршрутизация и компоновка страниц
* `widgets` — крупные композиционные блоки
* `features` — пользовательские сценарии
* `entities` — бизнес-сущности
* `shared` — переиспользуемые примитивы

Подробнее: [feature-sliced.design](https://feature-sliced.design/)
